# 调试状态日志模板

AI Agent **必须维护结构化调试日志**，显式追踪已尝试方案和结果，防止同质方案重复。

```
[调试日志] <问题摘要>
═══════════════════════════════════════
问题描述: {一句话描述}

已尝试的方案:
  #1: 假设H1=... → 验证=... → 结果=证伪 → 推断=排除原因A
  #2: 假设H2=... → 验证=... → 结果=证实 → 推断=原因B → 已修复
  #3: ...

当前范围:
  已排除: [模块X, 模块Y, ...]
  剩余可疑: [模块W, ...]
  当前最可能假设: {假设 + 依据}

下一步计划: {具体验证内容}
是否触发人工介入: 是/否（原因）
```

## 使用规则

- 开始调试**任何问题**前创建日志
- 每次尝试后更新（假设→验证→结果→推断）
- 进入下轮假设前回顾，确认新假设与已证伪的本质不同
- 日志不保存到文件，仅本轮工作记忆
- 问题解决后经验通过 `experience(save)` 持久化
