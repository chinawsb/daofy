### 8.14 尝试升级检查点

每次失败后的检查点，确保 3 次前有合理换档：

#### 第 1 次失败
```
□ 检查点 1
  ① 确认与上次是否本质不同（同质→立即换方向）
  ② 检查工具是否用对（§8.11 决策树）
  ③ 创建调试日志（§8.10）
  ④ 更新假设
```

#### 第 2 次失败
```
□ 检查点 2
  ① 查经验库（必须有！）
  ② 评估二分法隔离
  ③ 尝试不同工具/方法（同工具2次→换方向）
  ④ 更新已排除范围+剩余可疑
```

#### 第 3 次失败（介入前）
```
□ 检查点 3 — 介入前置检查
  ① 调试日志完整性
  ② 经验搜索已执行
  ③ 异常现场信息完整（§8.3）
  ④ 二分法已被尝试
  ⑤ 按 §8.4 填写介入申请书
  ✅ 全部满足 → 提交
  ❌ 任何不满足 → 补齐后再申请
```

#### 升级流程总图
```
尝试#1 → 同质?→立即换方向
        → 检查点1 → 更新日志 → 更新假设 → #2
尝试#2 → 查经验库+二分法评估
        → 换工具? → 检查点2 → 更新日志 → 换方向 → #3
尝试#3 → 检查点3(介入前置)
        ├─ 通过 → §8.4 申请介入
        └─ 不通过 → 补齐后重试
```

**关键规则**：
- 第1和第2次之间必须换方向
- 第2和第3次之间必须查经验库+评估二分法
- 同质方案立即触发 P2 换方向
- 严格区分"尝试计数"和"方案计数"
